Platform Overview

Fortnite esports platform with island generation, live channels, social features, and authenticated player dashboards.

What is live now

  • Epic Games OAuth flow is wired and ready for approved production keys.
  • Island Forge generates UEFN terrain from uploaded audio analysis and writes each run into a named output folder with PNG, JSON, manifest, and Verse package artifacts.
  • Channels provides a multi-category embedded guide with suggestions.
  • Arena now has a dedicated A-Frame spectator deck at /arena with leaderboard, theater, and ops-rail concepts.
  • Community and feed routes support social posts, updates, and member identity.
  • Oracle Autonomous Database stores members, posts, islands, and channels.
  • OCI Object Storage serves public media and generated assets.
◈
Status

Live at triptokforge.org. The platform is operating on Oracle now; Epic brand approval is the remaining gate for final production login credentials.

02

System Architecture

Oracle Always Free ARM running nginx, Gunicorn, Flask blueprints, Oracle DB, and OCI-backed media.

Infra map

  • VM: Oracle Always Free ARM on Ubuntu 22.04.
  • Proxy: nginx forwards public traffic to Gunicorn on 127.0.0.1:5000.
  • App: Flask blueprints deployed through systemd service islandforge.
  • Database: Oracle Autonomous with thin-mode python-oracledb.
  • Storage: OCI bucket triptokforge in us-ashburn-1.
  • Repo: github.com/iamjedi888/ver-perlinforge on branch main.

Blueprint structure

  • routes/platform.py handles home, forge, gallery, feed, community, admin, and health.
  • routes/api.py handles social APIs such as posts and likes.
  • routes/channels.py serves channels and channel suggestions.
  • routes/auth.py manages Epic OAuth setup and callback flow.
  • routes/whitepages.py serves the internal docs and track feed endpoints.

Deploy Guide

Push to GitHub main, pull on Oracle, restart the service, and verify health locally on the box.

01
Push the change

git add .
git commit -m "message"
git push origin main

02
Pull on Oracle

cd ~/ver-perlinforge/islandforge
git pull origin main

03
Restart and verify

sudo systemctl daemon-reload
sudo systemctl restart islandforge
sudo systemctl status islandforge --no-pager
curl http://127.0.0.1:5000/health

⚠
Deploy rule

Do not use git reset --hard on the Oracle instance for routine deploys. Treat the server like a pull target for origin/main.

What stays canonical

  • README.md for GitHub entry context.

  • The live runtime code, templates, static assets, and route modules under islandforge/.

What was identified as drift

  • Tracked patch scripts duplicated across the repo and across branch snapshots.
  • Old txt deploy notes that describe obsolete file-replacement workflows instead of the current GitHub → Oracle deploy path.
  • A tracked scratch file with inline secrets, which should never remain in version control.
  • Nested duplicate app copies such as islandforge/Island Forge 2/ and backup files like routes/forge.py.backup.

Security note

If any secrets from historical scratch notes were ever real, rotate them. WhitePages should document secret handling patterns, not contain live secret values.

Verse Architecture

Tarkov-Lite systems in Verse for UEFN using immutable player data and stateful UI overlays.

Core patterns

  • Immutable data: build fresh player records instead of mutating shared state.
  • UI state: store HUD references per player for reliable cleanup and redraw.
  • Material encoding: compress rarity and element into deterministic inventory slots.
  • Persistence: keep long-term player progression compatible with seasonal updates.
  • Death penalty: preserve risk and extraction tension as a core loop.
  • World layout: keep the 32 farm plots in one coastal social cluster near town, then push biome escalation outward.

Island Forge

Audio-to-heightmap and Verse package workflow for UEFN island generation.

01
Upload + analyze

Upload a track in Forge. The audio pass extracts sub-bass, bass, midrange, presence, brilliance, BPM, and duration, then maps those values into terrain signals.

02
Set scale + theme

Choose world size, water, wrap, farm direction, cluster density, and Fortnite chapter theme before you generate. Large worlds change the UEFN import procedure because World Partition becomes mandatory.

03
Generate run

Each run is saved locally as outputs/WorldName#N/ with heightmap.png, preview.png, layout.json, manifest.json, generated Verse files, and a verse_package.zip when available.

04
Import to UEFN

Use the saved run folder as the handoff bundle. Import the heightmap, follow the World Partition note when the chosen size requires it, then place the generated Verse files and bind builtin Fortnite assets in-editor.

Current target layout keeps all 32 farm plots in one shared coastal hub near town, with coherent Fortnite prefab kits assigned biome by biome instead of random prop mixing.

The important contract is now explicit: changing world scale changes the downstream UEFN steps. If Forge flags World Partition, the player needs to enable it before import instead of trying to brute-force the landscape into a default island.

⚠
Import rule

If the selected world scale exceeds roughly 2km, enable World Partition and Streaming in UEFN before importing the generated heightmap. The same warning is repeated in Forge, layout.json, and the generated package README.

Forge Runbook

  1. Open /forge, upload an audio track, and let the analysis complete so the terrain weights reflect the real file instead of defaults.
  2. Name the world and choose the Fortnite chapter theme you want the asset manifest to target.
  3. Set world scale, resolution, water level, wrap, farm direction, and cluster density. If the Forge World Partition flag appears, plan to enable World Partition in UEFN before import.
  4. Press Generate. Confirm the run saved into outputs/WorldName#N/ and download or open the generated files from the Forge locker.
  5. In UEFN, create or open the target island. For large worlds: Edit -> Project Settings -> World -> Enable World Partition, then World Settings -> Enable Streaming.
  6. Import heightmap.png as the landscape using the recommended resolution from Forge. Use preview.png as a visual reference and layout.json for metadata, town center, and plot positions.
  7. Copy the generated Verse files into your project or extract verse_package.zip. Keep the generated files together with asset_manifest.json and landscape_config.json.
  8. Bind the generated Verse slots to builtin Fortnite gallery and prop assets inside UEFN. The generated package documents supported builtin content-browser hints; it does not require custom shipped assets.
  9. Launch a session, validate streaming cells, town placement, plot spacing, and coastline readability, then iterate by regenerating the next numbered run folder instead of overwriting the old one.

Tarkov-Lite Vision

Persistent extraction MMO inside Fortnite with risk, insurance, progression, and evolving world systems.

Players stage from a safe hub, run extraction zones for materials and kills, and decide whether to bank progress or press deeper for better loot.

Economy loop

  • Grow or gather materials, then sell or craft around rarity and element value.
  • Kill enemies to earn drops, XP, and quest progression.
  • Extract safely to keep the run. Die and the carry risk becomes real.
  • Use insurance tokens to protect the most important item in a run.
  • Level up into harder zones, more systems, and higher-end cosmetics.
15

Game Systems

Core gameplay systems that define the extraction loop and long-term progression.

Death penalty

  • Lower stacks create full-loss pressure and teach early caution.
  • Higher-stack players still take meaningful loss while keeping late-game progression viable.
  • Insurance creates decision tension instead of full safety.

Material economy

  • Five primary material families keep collection readable.
  • Eighteen elements and multiple rarity bands support broad crafting depth.
  • A deterministic slot and value model keeps UI and storage simpler.
